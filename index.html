CSS Values and Units - what are they ?

In CSS specifications websites and on the property pages on MDN one can easily spot value types as they will be surrounded by angle brackets, such as <color> or <length>. When you see the value type <color> as valid for a particular property, that means you can use any valid color as a value for that property.

Values in CSS

A value type in CSS is a way to define a collection of allowable values. If we take the value type of <color> - you don't need to wonder which of the different types of color value can be used — keywords, hex values, rgb() functions, etc. You can use any available <color> values, assuming they are supported by your browser.

Units in CSS

What then is a CSS unit? Effectively, it is the unit of measurement that you use to determine the size of a particular property which you are styling for a specific element, group of elements or the container of the element.

For example:

          
            h2 { 
              padding: 2em; 
              }

If we look at the above ruleset, we can see that the property we are styling for this h2 selector is its padding.

The value we are using is 2em, - the unit in question is em, which is a relative unit as we will see below.

Type, class, and ID selectors

Data Type Description Example
<integer> An <integer> is a whole number. 1, 1024 or -11.
<number> A <number> represents a decimal number - it might contain a decimal point with a fractional component - then again it may not. 0.435, 138 (imagine the .0), or -1.2.
<dimension> A <dimension> is a <number> which has a unit attached to it. It is an umbrella category which includes <length>, <angle>, <time>, and <resolution> types.
<percentage> A <percentage> represents a fraction of some other value. For example, 50%. Percentage values are always relative to another quantity. For example, an element's length is relative to its parent element's length. 50%

Lengths

The numeric type you will come across most frequently is <length>. For example 10px (pixels) or 30em. There are two types of lengths used in CSS — relative and absolute. It's important to know the difference in order to understand how big things will become.

Absolute Length

The following are all absolute length units — they are not relative to anything else, and are generally considered to always be the same size.

Unit Name Equivalent to
cm Centimeters 1cm = 38px = 25/64in.
mm Millimeters 1mm = 1/10th of 1cm
Q Quarter-millimeters 1Q = 1/40th of 1cm
in Inches 1in = 2.54cm = 96px
pc Picas 1pc = 1/6th of 1in
pts Points 1pt = 1/72th of 1in
px Pixels 1px = 1/96th of 1in

Most of these units are more useful when used for print, rather than screen output. For example, we don't typically use cm (centimeters) on screen. The only value that you will commonly use is px (pixels).

Relative length units

Relative length units - as the name implies are relative to something else, which could be the size of the parent element's font, or the size of the viewport. The point of using relative units is that via careful planning you can make it so the size of text or other elements scales relative to everything else on the page. i.e. a step towards responsive web development. Some of the most useful units for web development are:

Unit What it's relative to:
em Font size of the parent, in the case of typographical properties like font-size, and font size of the element itself, in the case of other properties like width.
ex x-height of the element's font.
ch The advance measure (width) of the glyph "0" of the element's font.
rem Font size of the root element.
lh Line height of the element
vw 1% of the viewport's width'.
vh 1% of the viewport's height.
vmin 1% of the viewport's minimum dimension.
vmax 1% of the viewport's maximum dimension.
